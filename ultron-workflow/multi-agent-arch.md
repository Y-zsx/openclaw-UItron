# 多智能体协作架构设计

## 愿景
构建一个自主、高效、可扩展的多智能体系统，让奥创能够同时处理多个任务。

## 核心组件

### 1. 主脑 (Orchestrator)
- 负责任务分发与协调
- 维护全局状态 (state.json)
- 决策任务优先级

### 2. 专业Agent类型

| Agent | 功能 | 实现方式 |
|-------|------|----------|
| 监听Agent | 监控天气/邮件/日历 | Cron + heartbeat |
| 执行Agent | 浏览器自动化/文件处理 | sessions_spawn |
| 分析Agent | 数据分析/决策 | 主脑直接处理 |
| 通信Agent | 消息路由/通知分发 | message tool |

### 3. 通信机制

```
主脑 → state.json (写入任务)
  ↓
Agent → 读取state.json (领取任务)
  ↓
执行 → 更新state.json (报告结果)
  ↓
主脑 → 验证 → 决定下一步
```

### 4. 任务流

1. **任务下发**: 主脑根据状态决定任务
2. **Agent领取**: Sub-agent通过spawn获取任务
3. **执行报告**: 写入memory/或更新state
4. **结果验证**: 主脑验证并更新进度

## 实现路径

1. 定义Agent接口规范
2. 实现第一个专业Agent(监听Agent)
3. 建立任务队列机制
4. 添加失败重试与升级机制
