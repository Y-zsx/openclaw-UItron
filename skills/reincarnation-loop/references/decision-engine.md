# 决策引擎

## 核心决策流程

```
┌─────────────────────────────────────┐
│         唤醒 (Cron触发)              │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│  读取 ambition.md + state.json      │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│         决策节点 Check               │
└─────────────────┬───────────────────┘
                  ▼
    ┌─────────────┼─────────────┐
    ▼             ▼             ▼
有pending?    任务未完成?   有未完成里程碑?
    │             │             │
    Yes           Yes           Yes
    │             │             │
    ▼             ▼             ▼
执行        继续执行      选第一个里程碑
pending[0]  当前任务     开始推进
    │             │             │
    └─────────────┼─────────────┘
                  ▼
┌─────────────────────────────────────┐
│        都No → 标记完成，汇报          │
└─────────────────────────────────────┘
```

## 优先级规则

### 高优先级
1. **blocked任务** - 被阻塞的任务需要人工介入
2. **pending_tasks** - 待处理队列中的任务
3. **in_progress任务** - 正在进行中的任务

### 中优先级
4. **未完成里程碑** - 按顺序推进
5. **新任务** - 如果所有里程碑都完成但还想继续

### 低优先级（自主时间）
6. **学习/探索** - 了解相关新技术
7. **优化** - 改进现有代码

## 间隔策略

根据任务状态动态调整cron间隔：

| 状态 | 推荐间隔 |
|------|---------|
| 任务进行中 | 3-5分钟 |
| 等待外部 | 15-30分钟 |
| 探索学习 | 30-60分钟 |
| 即将完成 | 1-2分钟 |

## 决策示例

### 示例1: 任务进行中
```
state: {
  current_task: "实现登录模块",
  task_status: "in_progress"
}
→ 继续当前任务，间隔3分钟
→ 更新 this_life_accomplished
→ 更新 next_life_task 为 "实现订单模块"
```

### 示例2: 任务完成，需要推进
```
state: {
  current_task: "实现登录模块",
  task_status: "completed",
  next_life_task: "实现订单模块"
}
→ 设置 current_task = "实现订单模块"
→ task_status = "in_progress"
→ 继续执行
```

### 示例3: 全部完成
```
state: {
  current_task: "集成下单功能",
  task_status: "completed",
  history: [5条记录]
}
→ 更新 ambition_status = "completed"
→ 汇报完整结果给用户
→ 停止cron
```

## 关键原则

1. **每次都要推进** - 绝不能做重复劳动
2. **记录所有行动** - history是复盘的基础
3. **明确下一步** - next_life_task让系统知道方向
4. **适时停止** - 达成目标主动汇报，不恋战
