# 夙愿文档模板

## 结构

```markdown
# [项目/目标名称]

## 当前目标
一句话描述当前要完成的核心任务。

## 背景（可选）
为什么做这个？有什么约束？

## 里程碑
- [ ] 里程碑1：具体描述
- [ ] 里程碑2：具体描述
- [ ] 里程碑3：具体描述

## 验收标准
- 标准1：具体可衡量的条件
- 标准2：具体可衡量的条件

## 当前进度
[简要描述已完成了什么]
```

## 示例

```markdown
# 自动化薅羊毛系统

## 当前目标
搭建一个自动监控京东优惠商品并下单的系统

## 背景
- 需要多平台比价
- 需要自动登录和下单
- 需要处理验证码

## 里程碑
- [ ] 调研京东联盟API
- [ ] 搭建商品监控脚本
- [ ] 实现自动登录模块
- [ ] 集成下单功能
- [ ] 测试完整流程

## 验收标准
- 系统能自动发现100元以下的优惠商品
- 自动下单成功率>80%
- 每日报告发送至钉钉

## 当前进度
- 已调研3个电商平台API
- 完成监控脚本初版
```

## 关键点

1. **目标要具体** - "完成某事"太模糊，"在3天内搭建一个能监控X的系统"更清晰
2. **里程碑要可验证** - 每个里程碑都应该能判断是否完成
3. **验收标准要量化** - "系统稳定"不行，"99.9%可用"可以
