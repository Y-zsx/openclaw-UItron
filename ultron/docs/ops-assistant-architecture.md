## 第25世: 基础架构设计

### 1. 系统架构概览

```
┌─────────────────────────────────────────────────────────────────┐
│                    智能运维助手 (OpsAssistant)                   │
├─────────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │  监控采集层   │→ │  数据处理层   │→ │  智能分析层   │          │
│  │  Collectors  │  │  Processors  │  │  Analytics   │          │
│  └──────────────┘  └──────────────┘  └──────────────┘          │
│         ↓                ↓                ↓                     │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │  告警通知层   │← │  决策引擎层   │← │  自动修复层   │          │
```

### 2. 监控指标体系

| 类别 | 指标 | 阈值建议 | 采集频率 |
|------|------|----------|----------|
| CPU | 使用率、负载、进程数 | >80% / >核心数 | 30s |
| 内存 | 使用率、可用量、Swap | >85% / <100MB | 30s |
| 磁盘 | 使用率、IO等待、 inode | >90% / >20ms | 60s |
| 网络 | 流量、连接数、丢包率 | 动态基线 | 30s |
| 进程 | CPU/内存占用、状态 | 异常退出 | 60s |
| 服务 | 端口可用性、响应时间 | 超时/不可达 | 60s |

### 4. 核心模块设计

```python
# ops_assistant.py - 智能运维助手主类
class OpsAssistant:
    - collectors: 监控数据采集器
    - analyzer: 智能分析引擎
    - notifier: 告警通知器
    - auto_fixer: 自动修复模块
    
    + start()           # 启动服务
    + stop()            # 停止服务
    + get_status()      # 获取状态
    + run_diagnosis()   # 手动诊断
```

### 5. 告警分级

| 级别 | 名称 | 触发条件 | 通知方式 |
|------|------|----------|----------|
| P1 | 紧急 | 服务不可用、数据丢失风险 | 即时电话+消息 |
| P2 | 高 | 资源>90%、核心指标异常 | 消息+邮件 |
| P3 | 中 | 资源>80%、性能下降 | 消息 |
| P4 | 低 | 预警、趋势异常 | 每日汇总 |
