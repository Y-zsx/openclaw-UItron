# 智能运维助手系统 - 架构文档

## 1. 系统架构 (5层)

```
┌─────────────────────────────────────┐
│         展示层 (Dashboard)          │
├─────────────────────────────────────┤
│         告警层 (Alerting)           │
├─────────────────────────────────────┤
│         分析层 (Analysis)           │
├─────────────────────────────────────┤
│         处理层 (Processing)         │
├─────────────────────────────────────┤
│         采集层 (Collection)         │
└─────────────────────────────────────┘
```

## 2. 监控指标体系

| 类别 | 指标 | 采集方式 |
|------|------|----------|
| CPU | 使用率、负载、上下文切换 | psutil |
| 内存 | 使用率、可用、缓存 | psutil |
| 磁盘 | 使用率、IO读写 | psutil |
| 网络 | 流量、连接数 | psutil |
| 进程 | CPU/内存占用TOP10 | psutil |
| 服务 | 进程存活、端口响应 | subprocess |

## 3. 数据存储

- **时序数据**: InfluxDB / SQLite (轻量)
- **告警记录**: MySQL / SQLite
- **日志**: 文件存储 + 日志轮转

## 4. 告警分级

| 级别 | 说明 | 通知策略 |
|------|------|----------|
| P1 | 严重故障 | 即时通知 (钉钉/短信) |
| P2 | 重要异常 | 5分钟内通知 |
| P3 | 警告 | 汇总通知 |
| P4 | 提醒 | 每日汇总 |

## 5. 采集器设计

```python
# 采集器核心类
class MetricCollector:
    - cpu_metrics()    → CPU指标
    - memory_metrics() → 内存指标
    - disk_metrics()   → 磁盘指标
    - network_metrics()→ 网络指标
    - process_metrics()→ 进程TOP10
    - service_checks() → 服务健康检查
```
